1. 反射

1.1 初识反射

需求:通过外部文件配置,在不修改源码来控制程序
//==================传统使用=============================
传统方式==>反射机制
--Properties类能获取配置类的key value
    Properties properties = new Properties();
    properties.load(new FileInputStream("src\\main\\resources\\re.properties"));
            String classfullpath = properties.get("classfullpath").toString();
            String method = properties.get("method").toString();
            System.out.println("classfullpath="+classfullpath);
            System.out.println("method="+method);
            //虽然能获取key value但是字符串无法直接创建对象
//==================反射机制=============================       
            /** 不修改源码情况下,外部文件进行功能实现
             *(1) 加载类,返回Class类型的对象cls
             *(2) 通过cls得到你加载的类的对象实例
             *(3) 通过cls可以得到方法对象:在反射中,可以把方法视为对象(万物皆对象)
             *(4) 通过Method 来调用方法:即通过方法对象来实现调用方法
             **/
            Class<?> cls = Class.forName(classfullpath);
            Object o = cls.newInstance();
            System.out.println(o.getClass()); //运行类型
            Method method1 = cls.getMethod(method);
            method1.invoke(o);
//传统:对象.方法() , 反射机制 方法.invoke(对象)
 //==================初识API=============================  
             /**
              *  getField：只能获取公共的类成员变量
              *  getConstructor:获取指定构造器根据传参决定
              **/
             Field age = cls.getField("age");
             System.out.println(age.get(o));
      //传统写法:对象.成员变量  反射:成员变量.get(对象)
Constructor<?> constructor = cls.getConstructor();
Constructor<?> constructor2 = cls.getConstructor(String.class,Integer.class);

1.2 反射机制

Java Reflection
    1.反射机制允许程序在执行期间借助于Reflection API取得任何类的内部消息
      (比如成员变量,构造器,成员方法等等),并能操作对象的属性及方法。反射在
      设计模式和框架底层都会用到
    
    2.加载完类之后,在堆中就产生了一个Class类型的对象(一个类只有一个Class对象),
      这个对象包含了类的完整结构信息。通过这个对象得到类的结构。这个对象就像一面
      镜子，通过这个镜子看到类的结构,所以形象的称之为:反射。

1.2.1 java计算机的3个阶段

1. (代码)编译阶段:java程序通过javac编译成class字节码文件 
===>类加载器ClassLoader: class字节码-> Class类对象
2. (Class类阶段)加载阶段:在堆中生成一个Class类对象,拥有与对象对应的所有属性
<===> 双面映射对象知道它属于那个Class对象
3. runtime运行阶段:当调用时,在堆中生成对象或者使用Class对象
    
--  java反射机制可以完成
    在运行时判断任意一个对象所属的类
    在运行时构造任意一个类的对象
    在运行时得到任意一个类所具有的成员变量和方法
    在运行时调用任意一个对象的成员变量和方法
    生成动态代理
--  java.lang.reflect:位置

1.2.2 反射优缺点及优化

优点--可以动态创建和使用对象(框架底层核心,使用灵活)
缺点--使用反射基本时解释执行,对执行速度有影响
优化--关闭访问检查
    Method和Field、Constructor对象都有SetAccessible()方法
    SetAccessible作用时启动和禁用访问安全检查的开关
    参数值为true表示取消访问坚持
    SetAccessible方法继承AccessibleObject

1.3 Class类对象分析

1.3.1 基本介绍

1.Class也是一个类继承与Object
2.Class类对象不是new出来的,而是系统创建的(ClassLoader的load方法)
3.对于某个类的Class对象,在内存中只存在一份,因为类只加载一次
4.Class对象存放在堆中
5.通过Class对象可以完整得到一个类的完整结构
6.类的字节码二进制数据,是放在方法区(类的元数据:包括变量名,方法名,访问权限)
    --引用到对应的数据结构 Class类对象/堆

1.3.2 常用方法

// ?不确定的java类型
Class<?> cls = Class.forName(classAllPath);
    cls // 显示cls,是哪个类的Class对象。class com.lml.refCommonMethod.Cat
    cls.getClass() // 显示cls运行类型 class java.lang.Class
// 获取包名  com.lml.refCommonMethod      
cls.getPackage().getName() 
// 全类名 com.lml.refCommonMethod.Cat    
cls.getName()  
// 创建对象实例
Cat cat = (Cat)cls.newInstance();  
// 获取属性值 不能获取私有       
Field brand = cls.getField("brand");
brand.get(cat) 
// 给属性赋值        
brand.set(cat,"奔驰");
//获取所有属性字段
Field[] fields = cls.getFields();
    for (Field field : fields) {
        System.out.println(field.getName);
    }

1.3.3 获取Class类对象的6种方式

//多用于配置文件,读取类全路径,加载类
1.编译期:Class.forName(classAllPath)
    
//多用于参数传递,构造器
2.类加载阶段:类名.class
  
//已有对象实例
3.new Cat().getClass();

//通过类加载器[4种ClassLoader]来获取
4.ClassLoader classLoader = new Cat().getClass().getClassLoader();
        Class<?> cla4 = classLoader.loadClass(classAllPath);

//5.基本类型的Class对象
Class<Integer> integerClass = int.class;

//6.包装类.TYPE
 Class<Integer> type = Integer.TYPE;

1.3.4 Class对象范围

1.外部类,成员内部类,静态内部类,局部内部类,匿名内部类
2.interfface:接口
3.数组
4.enum:枚举
5.annotation:注解
6.基本数据类型与包装类
7.void

1.4 类加载

1.4.1 ClassLoad加载时期与类型

ClassLoad
反射机制是java实现动态语言的关键,也就是通过反射实现类动态加载。
1.静态加载: 编译时加载相关的类,如果没有则报错,依赖性太强。
    --直接使用实例对象
2.动态加载: 运行时加载需要的类,如果运行时不用该类,则不报错,降低了依赖性。
    --使用反射、
    
ClassLoad加载时期
    1.new:静态
    2.当子类被加载时,父类也加载:静态
    3.调用类中的静态成员时:静态
    4.通过反射:动态

1.4.2 类加载过程初解

类加载三个阶段
    1.加载 Loading
    2.连接 Linking
        (1):验证 verification 对文件安全,格式,元数据,字节码,符号引用
        (2):准备 preparation 对静态变量的默认初始化
        (3):解析 resolution 对常量池的符号引用替换成直接引用
    3.初始化 initialization

类加载各阶段完成任务

1.4.3 类加载过程详解

加载阶段 Loading

JVM 在该阶段的主要目的是将字节码从不同的数据源(可能是class、也可能是jar包、甚至网络)转化为二进制字节流加载到内存中,并生成一个代表该类的java.lang.Class对象。
主要在方法区生成类的字节码二进制数据和在堆区中生成Class对象。

连接阶段 Linking

1.验证:
    1.目的是为了确保Class文件的字节流包含的信息符合当前虚拟机的要求,并且不会危害虚拟机自身的安全。
    2.包含:文件格式验证(是否以魔数 oxcafebabe开头)、元数据验证、字节码验证和符号引用验证
    3.可以考虑使用 -Xverify:none 参数来关闭大部分的类验证措施,缩短虚拟机类加载的时间。
    
2.准备:
    1.JVM会在该阶段对静态变量,分配内存并默认初始化(对应数据类型的默认初始值,如0、0L、null、false等)。这些变量所使用的内存将在方法区中进行分配。
        static class  A{
        // n1 是实例属性,不是静态变量,因此在准备阶段,不会分配内存
        public int n1 = 10;
        // n2 是静态变量,分配内存 n2 准备阶段默认初始化0，而不是20(初始化阶段赋值)
        public static int n2 = 20;
        // n3 是final 是常量,会在准备阶段初始化为30
        public static final int n3 = 30;
    }

3.解析:
    1.虚拟机将常量池内的符号引用替换为直接引用的过程。

初始化 initialization

1.到初始化阶段,才真正开始执行类中定义的java程序代码,此阶段是执行<clinit>()方法的过程。
2.<clinit>()方法是由编译器按语句在源代码中出现的顺序,依次自动收集类中的所有[静态变量]的赋值动作和[静态代码块]中的语句,并进行合并。
3.虚拟机会保证一个类的<clinit>()方法在多线程环境中被正确地加锁、同步,如果多个线程同时去初始化一个类,那么只会有一个线程去执行一个类地<clinit>()方法,其他线程都需要阻塞等待,直到活动线程执行<clinit>()方法完毕。
    --源码里有个加锁机制,正因为这样,才保证了某个类在内存中只存在一份。

1.6 反射操作对象

1.6.1 反射创建对象[2种]

1.调用类中的public修饰的无参构造器
2.调用类中的指定构造器
    --Class类相关方法
        --newInstance:调用类中的无参构造器,获取对应类的对象。
        --getConstructor(Classs...clazz):根据参数列表,获取对应的构造器对象
        --getDecalaredConstructor(Classs...clazz):根据参数列表,获取对应的构造器对象
    --Constructor类
        --SetAccessible
        --newInstance(Object...obj):调用构造器
//获取所有公共构造器
Constructor<?>[] constructors = userClass.getConstructors();
//获取构造器参数个数
constructor.getParameterCount()
//构造器创建对象:调用类中的public修饰的无参构造器或直接使用Class类的newInstance()  
constructor.newInstance()
    
//有参构造找到对应类型的有参构造器
userClass.getDeclaredConstructor(Integer.class,String.class);
//强制爆破私有构造器
declaredConstructor.setAccessible(true);
//有参构造器创建对象
declaredConstructor.newInstance(100000, "1万年");

1.6.2 反射操作属性

1.Class对象.getDeclaredField(属性名)
2.setAccessible(true);
3.访问:
    cls.set(obj,值);//设置属性值 obj:对象
    cls.get(obj); //返回属性值
4.如静态属性,set和get的参数obj,可以写成null

1.6.3 反射操作方法

//获取指定长度的对应方法
1.cls.getDeclaredMethod(方法名,参数.class ...) 
//调用方法 指定参数值
2.cls.invoke(obj,参数...);  
//如是静态方法 obj对象 可传null
